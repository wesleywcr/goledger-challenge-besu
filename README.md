# GoLedger Challenge - Besu Edition

## Project Overview

This project implements a technical integration challenge between three layers:

- A private blockchain network with **Hyperledger Besu**;
- A **SimpleStorage** smart contract in Solidity;
- An API in **Go** that communicates with the blockchain and a **PostgreSQL** database.

The implementation of the solution is concentrated in `app/`, while `besu/` handles the local network and `SimpleStorage/` handles the contract and deployment with Foundry.

We recommend a UNIX-like machine (Linux/macOS).

## Endpoints REST

- `POST /value/set`
- `GET /value/get`
- `POST /value/sync`
- `GET /value/check`

## Operations
The application must implement 4 operations:
- **SET**: writes a value to the `SimpleStorage` contract;
- **GET**: reads the current value directly from the blockchain;
- **SYNC**: synchronizes the value from the blockchain to PostgreSQL;
- **CHECK**: compares the value in the database with the on-chain value.

## Set up the environment

The Makefile provides a single command to start the 4-node Besu network **and** deploy the `SimpleStorage` contract to it:

```bash
make devnet-deploy
```

The deployed contract address is printed to stdout at the end. You will need it in your application.

Other available targets:

| Target | Description |
|---|---|
| `make devnet` | Start the Besu network only |
| `make deploy` | Deploy the contract to a running network |
| `make stop-devnet` | Stop and clean up the network |

You can check the logs of a running node with:

```bash
docker logs -f besu.node-1
```

The contract ABI is generated by Foundry at `SimpleStorage/out/SimpleStorage.sol/SimpleStorage.json` after running `forge build` (or `make deploy`, which triggers a build automatically).

## How to run
1. Set up the Besu network and deploy the contract:

```bash
make devnet-deploy
```

2. Go to `app/`, copy the `.env` file and adjust the values:

```bash
cd app
```

3. Start the local PostgreSQL installation:

```bash
docker compose up -d
```

4. Execute the API:

```bash
go run .
```
# Technical Interaction between Besu, Go-Ethereum, and SQL

## Besu + Go-Ethereum

Even though it's Besu (a Hyperledger client), the JSON-RPC API is compatible with the Ethereum ecosystem. Therefore, the app uses the **go-ethereum** library to:

- connect via RPC (`ethclient`);
- sign transactions with a private key (`crypto`, `bind`);
- execute `eth_call` for reading;
- wait for transaction confirmation (`WaitMined`).

This coupling is possible because the EVM contract uses the same execution model.

## Go + SQL

SQL integration uses `database/sql` with the `pgx` driver:

- connection via DSN (the example in `.env.example` matches `app/docker-compose.yml`: user `db_besu_user`, database `db_besu`, port `5433`);
- initial schema created when the app starts (`CREATE TABLE IF NOT EXISTS`);
- idempotent write via upsert;
- deterministic read of a state row.

With this, the database acts as a local mirror of the on-chain value when the `SYNC` endpoint is called.

## Data Consistency

The system does not attempt to maintain automatic synchronization in the background. Instead:

- `GET` always reflects the blockchain in real time;
- `SYNC` updates the snapshot in the database;
- `CHECK` measures whether the snapshot and the current state still match.

This model simplifies the challenge and makes it clear when the divergence occurred.
